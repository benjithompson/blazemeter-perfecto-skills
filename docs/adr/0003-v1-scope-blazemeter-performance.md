# v1 scope: depth-first on BlazeMeter Performance Testing

For v1 we go depth-first on a single pillar — **BlazeMeter Performance Testing** — building a
coherent vertical of expert-workflow skills across its lifecycle, rather than one shallow skill per
pillar.

Rationale: Performance Testing has the most existing prior art (`bzm-analyze-test` plus the
`bzm-*` auth/artifact utilities) and the clearest expert-judgment value. Proving the skill pattern,
conventions, and quality bar deeply in one place yields a validated template that later pillars can
copy quickly.

Deferred to v2+ (each a later milestone replicating the pattern): BlazeMeter Virtual Services,
BlazeMeter API Test & Monitoring, and Perfecto. This is why a repo named for both platforms ships
only BlazeMeter Performance skills first — the breadth is sequenced, not abandoned.
