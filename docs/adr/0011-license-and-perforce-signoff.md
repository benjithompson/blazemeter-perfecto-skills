# License Apache-2.0; Perforce sign-off gates official branding and wide promotion

The repo is licensed **Apache-2.0** (permissive, with a patent grant and NOTICE convention suited to
corporate-adjacent tooling), with a `CONTRIBUTING.md` that points at the triage labels.

Constraint (not visible in the code): the repo wraps Perforce's commercial products and is published
from a personal GitHub account. Using **official BlazeMeter branding** and any **wide promotion** are
gated on Perforce internal sign-off. Until then we ship approximated branding (see ADR-0009) and keep
distribution low-key. Moving the repo under a Perforce-owned org is an open action; the
plugin/marketplace install path is location-independent, so the move is cheap.
